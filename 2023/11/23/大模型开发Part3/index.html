# Chat模型调用外部函数

## 3.1 Function Calling 功能产生背景

 尽管大模型由于其涌现能力的存在表现出十分强大的性能，当时在使用的过程中仍然会存在诸多的上限，例如无法获取最新的消息，无法获得一些具体的信息。OpenAI为了让用户在大模型的实际使用过程中能够有更高的上限，于2023年6月13日正式推出了function_calling功能，支持gpt模型通过调用外部函数来获取信息，从而达到让大模型输出更好结果的目的。

3.2 Function Calling功能使用

3.2.1 functions参数编写

 gpt模型通过ChatCompletion.create函数中的functions参数实现相关功能。该参数要求输入一个包含多个字典的列表，每个字典中包含一个外部函数的信息。字典要求包含的参数如下：

  • name：代表函数函数名称字的符串，必选参数，按照要求函数名称必须是 a-z、A-Z、0-9，或包含下划线和破折号，最大长度为 64。需要注意的是，name必须输入函数名称，而后续模型将根据函数名称在外部函数库中进行函数筛选；
  • description：用于描述函数功能的字符串，虽然是可选参数，但该参数传递的信息实际上是Chat模型对函数功能识别的核心依据。即Chat函数实际上是通过每个函数的description来判断当前函数的实际功能的，若要实现多个备选函数的智能挑选，则需要严谨详细的描述函数功能；（需要注意的是，在某些情况下，我们会通过其他函数标注本次对话特指的函数，此时模型就不会执行这个根据描述信息进行函数挑选的过程，此时是可以不设置description的。）
  • parameters：函数参数，必选参数，要求遵照JSON Schema格式进行输入，JSON Schema是一种特殊的JSON对象，专门用于验证JSON数据格式是否满足要求。

尝试为以上函数写一下functions参数 PS_SW_function = { 'name': "Paper_Search_ZW", "description": "对知网论文进行检索并保存", "parameters": {"type":"object", "properties":{ "search_key_dict":{ "type":"string", "description":"进行论文检索的关键词字典", } }, "required":["search_key_dict"], }, }

functions参数的编写确实比较难，尤其是其中的json schema对象，可以多看看上面的例子或者官方的说明文档进行学习。 https://json-schema.org/understanding-json-schema/

3.2.2 First Reponse

  • 完成函数调用共需要与大模型完成两次通信的过程，当前展示第一次通信。

 完成准备后开始第一次通信，需要注意的是function_call参数，共有三种取值

  1. 默认为None，表示不调用外部函数

  2. 取值为"auto"，表示模型自动选择是否调用

  3. 输入字典{"name": "my_function"}进行申明

3.2.3 Second Response

之后提取大模型要调用的函数名称和参数信息并在本地调用

 开始与大模型第二次通信，这次传回之前的调用申请和调用返回的结果，注意返回调用结果的角色应是"fucntion"。

可以看到，大模型较好的接收到了函数调用的生成结果并据此对问题做出了回答。至此，我们初步跑通了代码，使大模型具备了调用外部函数的能力！！！但在这个过程中还存在诸多问题，包括之前要解决的外部函数的缺点，以及返回结果体量太大的问题，这几个问题将会在后面尝试进行解决。
